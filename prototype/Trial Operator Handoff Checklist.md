# Trial Operator Handoff Checklist: Computer Use Verifier Kit

Source: [[UseDesktop as Verifier Infrastructure for Computer-Use Agents]]  
Package: [[Computer Use Verifier Kit]]

## Purpose
A one-page operator handoff that tells the next real trial runner exactly which existing artifacts to fill, in what order, without implying that validation has already happened.

## Trial sequence
1. Open [[Computer Use Verifier Kit/Computer Use Verifier Packet]] and define the real target, constraints, and success criteria.
2. During the run, paste every proof link into [[Computer Use Verifier Kit/Verifier Trial Evidence Index]] before editing any public claim.
3. After the run, fill [[Computer Use Verifier Kit/Verifier Promotion Decision Card]] from the proof, not memory.
4. Complete [[Computer Use Verifier Kit/Post-Trial Verifier Debrief Template]] and list which README, prototype, infographic, or skill-draft claims should change.
5. Only then patch the package README or skill draft; leave template-ready wording in place if evidence is incomplete.

## Minimum evidence checklist
- [ ] Workflow start state, end state, and pass/fail rubric captured.
- [ ] At least two agent attempts or one agent plus human baseline compared.
- [ ] Screenshots/logs/transcripts linked in the evidence index.
- [ ] Promote / pilot-only / iterate / hold decision filled from evidence.
